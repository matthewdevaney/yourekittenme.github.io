Deploying A Static Website To Github Pages

March 10, 2019 , Category: Web Development , Tags: github , git , website deployment

So you've built a static website and you want to share it with the world. Now you just need to get it up and running on the web. After doing some research you realize there are so many options for deployment and everyone has their own opinion. How do I choose the best method? Where can I find instructions? It all seems so overwhelming. At least that's how it felt for me.

There are a few reasons I like Github Pages for the task of hosting a small static website. It is purpose-built as a platform for developers to host their own blogs. I already use Github repositories to store the source code for my software projects so the deployment workflow comes naturally. Best of all it's free (as in beer).

In this tutorial I will show you a simple method for deploying your own static website and its source code to Github Pages. If you don't know how to build a static website and want to learn consider checking out Part 1 and Part 2 of my Pelican tutorial.

Prequisites

A basic understanding of the Git and Github is required. You will also need to sign up for a free Github account if you don't already have one.

Goals

  • Create Github repositories for your static website and its source code.
  • Go-live with your website

Getting Online

Its a good idea to keep your website and its source code in two separate folders.

blog
└── output
└── source

We are going to create two Github repositories: one to hold the generated website and another to hold its source code.

blog
└── output (repo: matthewdevaney.github.io)
└── source (repo: code_blog)

Generate the website using the command below.

pelican content -s publishconf.py

While in the output folder run the following git commands:

git init
git remote add origin https://github.com/yourekittenme/yourekittenme.github.io
git add -A
git commit -m "initial commit"
git push --set-upstream origin master

While in the source folder run the following git commands:

git init
git remote add origin https://github.com/yourekittenme/code_blog
git add -A
git commit -m "initial commit"
git push --set-upstream origin master
